Template includes

Includes are NOT the main way for template reuse in Django. You should use template inheritance first.

Includes are only used when you got a small piece of HTML that you know you will need several times, but not that often.

It works the way you expect :

<div>
    { % include 'app6_included.html' % }
</div>

<div>
    { % include 'app6_included.html' % }
</div>
will produce :
{% include 'app6_included.html' %}
{% include 'app6_included.html' %}

WARNING : the included template and the including template are rendered separatly, THEN merged. This means you can't access variables from one in another. If you need to do so, use with :

{ % include 'app6_included.html' with variable=value % }

Includes and inheritance can be used together without any problem.
